## 🧩 全局前端类比

> **Day 2 学的东西 ≈ 从 React 函数组件回到 Class Component，但加强100倍**

```typescript
// 前端你熟悉的"对象"——JSON一把梭
const user = {

  name: '小明',
```

```java
// Java的"对象"——必须先有类（模具），才能new出对象（产品）
```

# 📝 练习1：创建 User 类（含私有字段 + getter）

> **难度：⭐ | 核心概念：封装（Encapsulation）**

## 🧩 前端类比

```typescript
// 前端你熟悉的写法
class User {
  private id: number;
  private username: string;
  private email: string;
  
  constructor(id: number, username: string, email: string) {
    this.id = id;
    this.username = username;
    this.email = email;
  }
  
  // getter——像访问属性一样获取私有数据
  getId(): number { return this.id; }
  getUsername(): string { return this.username; }
  getEmail(): string { return this.email; }
}
```

## ☕ Java完整代码

```java
// 文件名: User.java
public class User {

    // ---------- 私有字段（private） ----------
    // 类比：React Class Component 里的 this.state，外部不能直接改
    private int id;
    private String username;
    private String email;

    // ---------- 构造器（Constructor） ----------
    // 类比：组件的 constructor(props) —— 初始化"状态"
    // 特点：1. 名字必须和类名一模一样  2. 没有返回类型（连void都不能写！）
    public User(int id, String username, String email) {
        this.id = id;
        this.username = username;
        this.email = email;
    }

    // ---------- Getter 方法 ----------
    // 为什么要有getter？因为private字段外部不能直接访问
    // 这在Java里叫"封装"——通过方法控制访问权限
    public int getId() {
        return id;        // 这里的 id 就是 this.id，不写 this 也可以（没有命名冲突时）
    }

    public String getUsername() {
        return username;
    }

    public String getEmail() {
        return email;
    }

    // ---------- 一个普通方法 ----------
    public void printInfo() {
        System.out.println("User{id=" + id + ", username='" + username + "', email='" + email + "'}");
    }
}
```

```java
// 文件名: UserTest.java（用来测试User类）
public class UserTest {
    public static void main(String[] args) {
        // 创建User对象 —— 就像 React 里 <User id={1} username="小明" email="xiao@163.com" />
        User user = new User(1, "小明", "xiao@163.com");
        
        // 通过getter获取数据
        System.out.println("ID: " + user.getId());
        System.out.println("用户名: " + user.getUsername());
        System.out.println("邮箱: " + user.getEmail());
        
        user.printInfo();
        
        // ❌ 下面这行会编译报错，因为 id 是 private 的
        // System.out.println(user.id);  // 报错：id has private access
    }
}
```


## ❓ 练习1的疑问点 & 难点（这里你会卡死！）

### 🔴 难点1：`this` 什么时候必须写？

```java
public User(int id, String username, String email) {
    // 这里参数名和字段名一模一样！
    // 如果不写 this，id = id 就是把参数id赋给参数id自己——没卵用
    this.id = id;         // ✅ this.id 是字段，id 是参数
    this.username = username;
    this.email = email;
}
```

> 🧩 **前端类比**：和TS/JS一模一样！`this.id = id` 表示"把参数id赋给实例的id属性"。如果参数名和字段名不同（比如 `public User(int userId, String name)`），就可以写成 `id = userId`，此时 `this` 可省略。

### 🔴 难点2：构造器为什么没有返回类型？

```java
public User(...) { }    // ✅ 构造器——没有返回类型
public void User(...) { }  // ❌ 这不是构造器！这是一个名叫User的普通void方法！
```

> **不要问为什么，这是Java语法规则。** 构造器的使命就是**初始化对象**，返回的是"这个刚建好的对象本身"——这个过程是JVM内部完成的，你不需要声明。

### 🔴 难点3：`private` 到底防谁？

```java
public class UserTest {
    public static void main(String[] args) {
        User user = new User(1, "小明", "xiao@163.com");
        
        // user.id    —— ❌ UserTest类不能访问User类的私有字段
        // 但是User类内部自己的方法可以随便访问自己的私有字段
    }
}
```

> 🧩 **前端类比**：就像ES2020的 `#privateField`——`user.#id` 在外面访问不到，只有类内部方法能访问。但Java的 `private` 是**编译期检查**而非运行时，比 JS 的 `#` 更早拦截错误。

### 🔴 难点4：为什么非要getter？直接 `public` 不行吗？

| 方式 | 代码 | 问题 |
|------|------|------|
| 直接暴露字段 | `public String email;` | 外部随便改：`user.email = "hacker@evil.com"` — 无法控制！|
| Getter封装 | `private String email;` + `public String getEmail()` | 你可以加校验逻辑、日志、甚至改内部实现 |

```java
// Getter 的威力——你可以在里面加逻辑！
public String getEmail() {
    // 比如：数据库里存的邮箱是小写的，返回时自动格式化
    return email.toLowerCase();  // 调用者无感知
}

// 如果直接 public，你后面想改格式——所有调用代码都得改！
```

> 🟢 **记忆口诀**：**"字段都private，方法看需求"。** 这是Java世界的黄金法则。

# 📝 练习2：创建 AdminUser（继承 User）

> **难度：⭐⭐ | 核心概念：继承（Inheritance） + super**

## 🧩 前端类比

```typescript
// 前端
class AdminUser extends User {
  private role: string;
  
  constructor(id: number, username: string, email: string, role: string) {
    super(id, username, email);  // 必须先调父类构造器！
    this.role = role;
  }
  
  getRole(): string { return this.role; }
}
```

## ☕ Java完整代码

```java
// 文件名: AdminUser.java
public class AdminUser extends User {   // ← extends = 继承

    // 子类新增的字段
    private String role;

    // 构造器：必须调用父类构造器 super(参数)
    public AdminUser(int id, String username, String email, String role) {
        super(id, username, email);   // ← 必须！调用父类User的构造器
        this.role = role;             // ← 然后初始化自己的字段
    }

    // 子类新增的方法
    public String getRole() {
        return role;
    }

    // 重写（Override）父类的 printInfo 方法
    @Override   // ← 注解：告诉编译器\"我是要重写父类方法\"
    public void printInfo() {
        // 先调用父类的实现——打印公共部分
        super.printInfo();
        // 再打印自己特有的部分
        System.out.println("  role='" + role + "'");
    }
}
```

```java
// 文件名: AdminUserTest.java
public class AdminUserTest {
    public static void main(String[] args) {
        AdminUser admin = new AdminUser(1, "管理员", "admin@myapp.com", "SUPER_ADMIN");
        
        // 继承来的方法可以直接用（不用再写一遍！）
        System.out.println("ID: " + admin.getId());           // 从User继承的getter
        System.out.println("用户名: " + admin.getUsername());  // 从User继承的getter
        System.out.println("角色: " + admin.getRole());        // AdminUser自己的方法
        
        admin.printInfo();  // 调用的是重写后的版本
    }
}
```

```bash
javac User.java AdminUser.java AdminUserTest.java && java AdminUserTest
```

### ✅ 预期输出
```
ID: 1
用户名: 管理员
角色: SUPER_ADMIN
User{id=1, username='管理员', email='admin@myapp.com'}
  role='SUPER_ADMIN'
```

## ❓ 练习2的疑问点 & 难点

### 🔴 难点1：`super(...)` 必须写在第一行！

```java
public AdminUser(int id, String username, String email, String role) {
    this.role = role;                   // ❌ 编译报错！
    super(id, username, email);         // super() 必须是第一行！
}
```

> **规则**：子类构造器里，`super()` 调用必须是**第一条语句**。为什么？因为父类的"零件"必须先初始化好，子类才能用。

### 🔴 难点2：如果父类没有无参构造器，子类必须显式调 `super(参数)`

```java
// User 类定义了构造器 User(int, String, String)
// —— 这意味着 Java不会自动生成默认无参构造器

// 所以 AdminUser 必须写 super(id, username, email)
// 如果 User 还有一个无参构造器 User()，那 AdminUser 可以不写super——Java会默认调super()
```

> 🧩 **前端类比**：就像React里，如果你的父组件props是必填的，子组件必须传过去。省略 `super(props)` → React报错。

### 🔴 难点3：`@Override` 不写行不行？

```java
// 不写 @Override 也能重写父类方法——但强烈建议写上！
public void printInfo() { ... }

// 写了 @Override 的好处：
// 1. 编译器会检查：父类到底有没有这个方法？
// 2. 如果你拼写错了（比如 printinfo），编译器直接报错
// 3. 代码可读性更好——一看就知道是重写
```

> 🟢 **记忆口诀**：**`@Override` ≈ 前端的 `// @ts-check`——帮编译器也帮你自己检查**

### 🔴 难点4：Java单继承的限制

```java
// ❌ Java不允许一个类继承多个父类
public class AdminUser extends User, SomeOtherClass { }  // 编译报错！

// ✅ 要解决这个问题，需要用接口（interface），Day 4会学到
```

> 🧩 **前端类比**：JS的 `class AdminUser extends User` 也是单继承。但JS有Mixin模式、组合模式来弥补。Java用 `interface` 来弥补——**更严格，但也更清晰**。

# 📝 练习3：带构造器重载的类

> **难度：⭐⭐⭐ | 核心概念：构造器重载（Overloading）**

## 🧩 前端类比

```typescript
// 前端：用默认参数来实现\"多种创建方式\"
class Product {
  constructor(
    public id: number,
    public name: string = '未命名',    // 默认值
    public price: number = 0           // 默认值
  ) {}
}

// 可以这样用
new Product(1);                // 只传id
new Product(1, '手机');         // 传id和name
new Product(1, '手机', 2999);   // 全传
```

## ☕ Java完整代码

```java
// 文件名: Product.java
public class Product {

    private int id;
    private String name;
    private double price;

    // ---------- 构造器重载：同一个方法名，不同参数列表 ----------
    
    // 1️⃣ 无参构造器
    public Product() {
        this.id = 0;
        this.name = "未命名商品";
        this.price = 0.0;
    }

    // 2️⃣ 只传id
    public Product(int id) {
        this.id = id;
        this.name = "未命名商品";    // 给默认值
        this.price = 0.0;
    }

    // 3️⃣ 传id和name
    public Product(int id, String name) {
        this.id = id;
        this.name = name;
        this.price = 0.0;
    }

    // 4️⃣ 全参构造器（全部字段都传）
    public Product(int id, String name, double price) {
        this.id = id;
        this.name = name;
        this.price = price;
    }

    // 5️⃣ 🔥 Java特有技巧：用 this() 调用另一个构造器（避免重复代码！）
    // 这个构造器和上面的无参构造器效果一样，但省去了重复赋值的代码
    public Product() {
        this(0, "未命名商品", 0.0);  // 调用第4个构造器！
    }
    // ⚠️ 注意：上面第1个和第5个不能同时存在，因为签名一样——会编译报错！
    // 实际开发中，你只需要保留其中一种写法

    // Getter
    public int getId() { return id; }
    public String getName() { return name; }
    public double getPrice() { return price; }

    public void printInfo() {
        System.out.println("Product{id=" + id + ", name='" + name + "', price=" + price + "}");
    }
}
```

```java
// 文件名: ProductTest.java
public class ProductTest {
    public static void main(String[] args) {
        // 4种方式创建对象
        Product p1 = new Product();                     // 无参
        Product p2 = new Product(1);                    // 只传id
        Product p3 = new Product(2, "手机");            // 传id+name
        Product p4 = new Product(3, "电脑", 5999.0);    // 全传

        p1.printInfo();
        p2.printInfo();
        p3.printInfo();
        p4.printInfo();
    }
}
```

```bash
javac Product.java ProductTest.java && java ProductTest
```

### ✅ 预期输出
```
Product{id=0, name='未命名商品', price=0.0}
Product{id=1, name='未命名商品', price=0.0}
Product{id=2, name='手机', price=0.0}
Product{id=3, name='电脑', price=5999.0}
```

## ❓ 练习3的疑问点 & 难点

### 🔴 难点1：Java没有"默认参数"——必须靠重载

```typescript
// 前端：优雅的默认参数
function createProduct(id: number, name = '未命名', price = 0) { }
```

```java
// Java：没有默认参数！只能通过重载（写多个构造器）或 Builder 模式
// 或者用 this() 调用另一个构造器来减少重复
```

> **这是从前端转Java最大的语法失落之一。** 你可以用重载 + `this()` 来模拟。到Spring Boot阶段会学 `@Builder` 注解来彻底解决。

### 🔴 难点2：`this()` 调用另一个构造器——也是第一行！

```java
public Product() {
    this(0, "未命名商品", 0.0);   // ✅ this() 必须是第一行
    // System.out.println("...");  // ❌ 不能在第一行写别的！
}
```

> `this()` 和 `super()` 一样，必须是**构造器的第一条语句**。两者不能同时存在（因为都必须是第一行）。

### 🔴 难点3：用 `this.xxx` 给字段赋默认值的两种方式

```java
// 方式A：在构造器里赋值
public Product() {
    this.id = 0;
    this.name = "未命名";
}

// 方式B：直接在字段声明处赋值（最简洁！）
public class Product {
    private int id = 0;              // ← 声明时直接给默认值
    private String name = "未命名";   // ← 声明时直接给默认值
    private double price = 0.0;
    
    public Product() { }  // 构造器里什么都不用写了
}
```

> **建议**：能用方式B就用方式B，代码更简洁。

### 🟡 避坑：构造器重载的调用规则

```java
Product p = new Product();      // 调用无参构造器
Product p = new Product(1);     // 调用(int)构造器
Product p = new Product(1, "a"); // 调用(int, String)构造器
Product p = new Product(1, "a", 99.0); // 调用(int, String, double)构造器
Product p = new Product(1, 99.0);  // ❌ 编译报错！没有(int, double)这个签名的构造器
```

# 📝 练习4：Order 和 OrderItem 关联关系

> **难度：⭐⭐⭐⭐ | 核心概念：对象关联 + 集合**

## 🧩 前端类比

```typescript
// 前端：订单包含多个订单项
interface OrderItem {
  productId: number;
  productName: string;
  quantity: number;
  price: number;
}

interface Order {
  id: number;
  customerName: string;
  items: OrderItem[];        // 一个订单有多个商品
  totalAmount: number;       // 总金额
}
```

## ☕ Java完整代码

### 第一步：先写 OrderItem（订单项）

```java
// 文件名: OrderItem.java
public class OrderItem {
    private int productId;
    private String productName;
    private int quantity;
    private double price;

    // 全参构造器
    public OrderItem(int productId, String productName, int quantity, double price) {
        this.productId = productId;
        this.productName = productName;
        this.quantity = quantity;
        this.price = price;
    }

    // Getter
    public int getProductId() { return productId; }
    public String getProductName() { return productName; }
    public int getQuantity() { return quantity; }
    public double getPrice() { return price; }

    // 计算这个小计：数量 × 单价
    public double getSubtotal() {
        return quantity * price;
    }

    public void printInfo() {
        System.out.println("  " + productName + " × " + quantity + " = ¥" + getSubtotal());
    }
}
```

### 第二步：再写 Order（订单——包含多个OrderItem）

```java
// 文件名: Order.java
import java.util.ArrayList;

public class Order {
    private int id;
    private String customerName;
    
    // 🔥 关键：Order 持有 OrderItem 的列表 —— "订单包含多个商品"
    // 前端类比：items: OrderItem[]
    private ArrayList<OrderItem> items;   // ArrayList 是 Java 的动态数组

    // 构造器
    public Order(int id, String customerName) {
        this.id = id;
        this.customerName = customerName;
        this.items = new ArrayList<>();   // 初始化空列表
    }

    // 添加一个商品到订单
    public void addItem(OrderItem item) {
        items.add(item);   // ArrayList.add() ≈ JS的 Array.push()
    }

    // 获取所有商品
    public ArrayList<OrderItem> getItems() {
        return items;
    }

    // 计算订单总金额
    public double getTotalAmount() {
        double total = 0;
        for (OrderItem item : items) {    // 增强for循环：遍历每个item
            total += item.getSubtotal();
        }
        return total;
    }

    // 打印订单详情
    public void printOrder() {
        System.out.println("===== 订单 #" + id + " =====");
        System.out.println("客户: " + customerName);
        System.out.println("商品列表:");
        
        for (OrderItem item : items) {
            item.printInfo();   // 委托给OrderItem自己去打印
        }
        
        System.out.println("总金额: ¥" + getTotalAmount());
        System.out.println("===================");
    }

    // Getter
    public int getId() { return id; }
    public String getCustomerName() { return customerName; }
}
```

```java
// 文件名: OrderTest.java
public class OrderTest {
    public static void main(String[] args) {
        // 创建一个订单
        Order order = new Order(1001, "小明");

        OrderItem item1 = new OrderItem(1, "iPhone 15", 1, 6999.0);
        OrderItem item2 = new OrderItem(2, "手机壳", 2, 39.0);
        OrderItem item3 = new OrderItem(3, "钢化膜", 3, 19.9);
        
        order.addItem(item1);
        order.addItem(item2);
        order.addItem(item3);
        
        // 打印订单
        order.printOrder();
        
        // 单独访问某个订单项
        System.out.println("\n第一个商品: " + order.getItems().get(0).getProductName());
    }
}
```

### 🔴🔴 难点1（最重要）：static 的精准语义

```java
public class IdGenerator {
    private static int nextId = 1;   // static变量
    // ↑ 它是"类级别的"，不是"实例级别的"
    
    // 内存中：这个 nextId 只有一份，存在"方法区"里
    // 不管 new 多少个 IdGenerator 对象，大家共享这一个 nextId
}
```

> 🧩 **前端类比**：
```typescript
// 前端准确的类比
class IdGenerator {
  // static ≈ 挂在 IdGenerator 构造函数本身上
  static nextId = 1;
  
  // 对比：实例字段挂在 this 上
  instanceField = 100;
}

const gen1 = new IdGenerator();

console.log(IdGenerator.nextId);  // 1 —— 类本身有

```

### 🔴 难点2：`static` 能访问什么，不能访问什么

```java
public class Demo {
    private static int staticVar = 1;
    private int instanceVar = 2;
    
    // ✅ 静态方法只能访问静态成员
    public static void staticMethod() {
        System.out.println(staticVar);     // ✅ 可以
        // System.out.println(instanceVar); // ❌ 编译报错！
        // this.instanceVar;                // ❌ 静态方法里没有 this！
    }
    
    // ✅ 实例方法可以访问静态成员和实例成员
    public void instanceMethod() {
        System.out.println(staticVar);     // ✅ 静态变量
        System.out.println(instanceVar);   // ✅ 实例变量
        System.out.println(this.instanceVar); // ✅
    }
}
```

> 🧩 **前端类比**：同样，React Class Component 的静态方法 `static getDerivedStateFromProps()` 里不能访问 `this.state`。

### 🔴 难点3：`static` 为什么不能调非 `static`？

```java
public class Test {
    public static void main(String[] args) {
        greet();  // ❌ 报错！main是static，不能调非static的greet()
    }
    
    public void greet() {
        System.out.println("Hi");
    }
}
```

**原因链**：
1. `main` 是 `static` → 不需要 `new Test()` 就能运行
2. `greet()` 不是 `static` → 必须 `new Test()` 才有"实例"去调
3. JVM 在运行 `main` 时还没创建任何 `Test` 实例
4. 所以 `main` 里不能直接调非 `static` 方法

**解决方案**：
```java
// 方案1：把 greet 也改成 static（推荐）
public static void greet() { System.out.println("Hi"); }

// 方案2：在 main 里 new 一个实例再调（Day 7常用）
public static void main(String[] args) {
    Test t = new Test();
    t.greet();
}
```

### 🟡 避坑：不要用 `实例.static方法()` 调用！

```java
IdGenerator gen = new IdGenerator();
gen.generateId();     // ✅ 能运行，但IDE会警告你！
// 应该用：IdGenerator.generateId();  // ✅ 正确姿势
```

> **静态成员应该通过类名访问，而不是实例。** 虽然Java允许你通过实例访问静态成员，但这是**迷惑行为**——IDE会画黄色波浪线警告你。

## 🔥 逐题分析：final 能用在哪儿？不能用在哪儿？

### 📝 练习1：User 类

```java
public class User {
    // ❌ id能不能加final？ → 看情况！
    // 如果id一旦创建就永远不变（比如数据库自增ID），那加final非常合理
    private final int id;        // ✅ 合理！用户ID创建后不变
    
    // ❓ username 和 email 加不加？
    private String username;     // ❌ 不能加final！用户可能改用户名
    private String email;        // ❌ 不能加final！用户可能改邮箱
    // 3个字段都加final → 这个对象就完全不可变了——"Immutable Object"

    public User(final int id, final String username, final String email) {
        this.id = id;
        this.username = username;
        this.email = email;
        // this.id = 999;  // ❌ 编译报错！final字段在构造器里只能赋值一次
    }
    
    // final方法？不必——User类本身没有被继承时，final方法没什么意义
}
```

### 🎯 练习1改造：Immutable User（不可变对象）

```java
// 文件名: ImmutableUser.java
public class ImmutableUser {

    private final int id;              // ✅ final：ID永不改变
    private final String username;     // ✅ final：创建后用户名不变
    private final String email;        // ✅ final：创建后邮箱不变

    // 构造器——所有字段一次性赋值，之后永远不能改
    public ImmutableUser(int id, String username, String email) {
        this.id = id;
        this.username = username;
        this.email = email;
    }

    // 只有getter，没有setter！这是"不可变"的关键
    public int getId() { return id; }
    public String getUsername() { return username; }
    public String getEmail() { return email; }

    public void printInfo() {
        System.out.println("ImmutableUser{id=" + id + ", username='" + username + "', email='" + email + "'}");
    }
    
    // 🔥 如果我要"修改"username怎么办？
    // 答：不能改！只能创建一个新的对象
    public ImmutableUser withUsername(String newUsername) {
        // 返回一个全新对象，原对象不变
        return new ImmutableUser(this.id, newUsername, this.email);
    }
}
```

```java
// 文件名: ImmutableUserTest.java
public class ImmutableUserTest {
    public static void main(String[] args) {
        ImmutableUser user = new ImmutableUser(1, "小明", "xiao@163.com");
        user.printInfo();
        
        // user.id = 2;          // ❌ 编译报错！final字段不能重新赋值
        // user.setUsername("xxx"); // ❌ 根本没有setter！
        
        // 想"改"？只能新创建一个
        ImmutableUser newUser = user.withUsername("大明");
        newUser.printInfo();     // id=1, username='大明'
        user.printInfo();        // id=1, username='小明' ← 原对象没变！
    }
}
```

> 🧩 **前端类比**：这不就是React的 `setState` 不可变更新的思路吗？
> ```jsx
> // React 里你不能直接改 state
> // ❌ this.state.name = '大明' 
> // ✅ this.setState({ name: '大明' })
> ```
>
> Java的不可变对象思路一模一样：**想改？那就创建一个新的。**

### 📝 练习2：AdminUser + final

```java
public class AdminUser extends User {
    
    // ❌ role 能不能加final？
    private final String role;  // ✅ 如果角色创建后不能变（比如一旦设定就是SUPER_ADMIN）
```

### 🎯 核心问题：`final` 类阻止继承

```java
// ❌ 如果我们把 User 声明为 final
public final class User {    // ← final类：禁止任何人继承我！
    // ...
}

// ❌ 编译报错！不能继承 final 类
public class AdminUser extends User {   // 错误: 无法从最终User进行继承
    // ...
}
```

> 🧩 **前端类比**：`final class` ≈ 你在JS/TS里有个类，但不想让别人 `extends`。只是JS没有原生机制阻止继承，Java有——它就是 `final`。

**什么样的类应该加 `final`？**
- `String` 类就是 `final` 的——你永远不能继承 `String`
- 工具类（如 `Math`、`Collections`）也是 `final` 的
- 安全敏感的类（防止子类篡改行为）

### 🔥 参数上加 `final`：神操作还是画蛇添足？

```java
// 你可以在方法参数上加 final
public User(final int id, final String username) {
    this.id = id;         // ✅ 可以读
    // id = 999;          // ❌ 编译报错！final参数不能重新赋值
    this.username = username;
}
```

```typescript
// 前端类比
function createUser(const id: number, const username: string) { 
    // TS不支持在参数上加const...但效果等价于：
    // 不重新赋值参数
}
```

> 🟢 **好处**：告诉编译器"这个方法里我不会手贱改参数"，减少bug。
> 🟡 **不写也行**：大多数Java工程师不在参数上加 `final`，因为默认就不该改参数（坏习惯）。如果你在方法里写了 `id = 999`，那说明设计有问题。

### 📝 练习3：构造器重载 + final

这里有个**最骚的操作**——用 `final` + `this()` 实现"终极不可变链式构造"：

```java
public class Product {
    private final int id;            // ✅ 全部final——对象完全不可变
    private final String name;
    private final double price;

    // 主构造器（private！不让外部直接调）
    private Product(int id, String name, double price) {
        this.id = id;
        this.name = name;
        this.price = price;
    }

    // 🔥 对外暴露的静态工厂方法（不用new，用静态方法创建）
    public static Product create(int id) {
        return new Product(id, "未命名商品", 0.0);
    }

    public static Product createFull(int id, String name, double price) {
        return new Product(id, name, price);
    }
}
```

```java
// 使用

Product p2 = Product.createFull(2, "手机", 2999.0);
```

> 🧩 **前端类比**：`Product.create(1)` ≈ `createProduct(1)` 工厂函数，比 `new Product(1)` 语义更清晰。

### 📝 练习4：Order + OrderItem + final

**这里 `final` 有一个非常经典的坑！**

```java
public class Order {
    private final int id;                  // ✅ ID不会变
    private final String customerName;     // ✅ 客户名不会变
    
    // ❓ items 能不能加 final？
    private final ArrayList<OrderItem> items;  // ✅ 可以加！但含义和你想象的不同！

    public Order(int id, String customerName) {
        this.id = id;
        this.customerName = customerName;
        this.items = new ArrayList<>();   // ← 这里赋值一次
    }
    
    public void addItem(OrderItem item) {
        items.add(item);   // ✅ 这能编译通过？！items不是final吗？？？
    }
}
```

### 🔴🔴🔴 最坑的考点：`final` 管的是"引用"还是"内容"？

```java
final ArrayList<OrderItem> items = new ArrayList<>();

// final 的意思是：items 这个**变量**不能再指向别的对象
items.add(new OrderItem(...));  // ✅ 可以！因为items指向的还是同一个ArrayList对象
items = new ArrayList<>();      // ❌ 编译报错！final变量不能重新赋值

// 类比：
final int x = 10;
x = 20;  // ❌ 编译报错

// 但对象就不同了——final管的是"箭头"指向谁，不管"箭头指向的对象内部怎么变"
```

> 🧩 **前端类比**：
> ```typescript
> const items: OrderItem[] = [];
> items.push({...});      // ✅ const 不阻止 push！
> items = [];             // ❌ 编译报错！const不能重新赋值
> ```

**关键结论：**
- `final` 对于**基本类型**（int, double等）→ 值不能变 ✅
- `final` 对于**引用类型**（ArrayList, String等）→ 引用（指针）不能变，但对象**内部可以变** ❌

### 📝 练习5：IdGenerator + final

```java
public class IdGenerator {
```

> ```
>
> 模式完全一样！只不过Java需要包在一个 `class` 里。

## 📊 总结表：final 用在哪 & 怎么用

| 场景 | 能加 `final` 吗 | 含义 | 前端类比 |
|------|----------------|------|----------|
| 字段 `private final int id` | ✅ | 构造器赋值后永不改变 | `readonly id: number` |
| 局部变量 `final int x = 5` | ✅ | 只能赋值一次，之后只读 | `const x = 5` |
| 方法参数 `public void foo(final int x)` | ✅ | 方法体内不能改参数 | 没有直接对应，但好习惯 |
| 方法 `public final void foo()` | ✅ | 子类不能重写 | — |
| 类 `public final class String` | ✅ | 不能被继承 | — |
| `static final` 组合 | ✅ | 全局常量 | `export const` |
| `ArrayList` 字段加 `final` | ⚠️ 可加 | 引用不能改，**但内容能改** | `const arr = []; arr.push(1)` ✅ |
| `static` 计数器加 `final` | ❌ **不能** | 加了就变不了值了 | 矛盾！ |
