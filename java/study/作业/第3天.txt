# 📝 练习1：把你JS项目中最复杂的数组操作，用Stream API重写

> **难度：⭐⭐⭐ | 核心概念：Stream API 的链式调用**

## 🧩 前端类比

假设你有一个前端常见的场景：从商品列表中筛选出价格大于100的、取前5个、按价格降序排序、只取名称和价格的映射。

### TypeScript 版本（你最熟悉的）

```typescript
// 商品类型
interface Product {
  id: number;
  name: string;
  price: number;
  category: string;
  stock: number;
}

const products: Product[] = [
  { id: 1, name: "iPhone", price: 6999, category: "手机", stock: 10 },
  { id: 2, name: "华为Mate", price: 5999, category: "手机", stock: 5 },
  { id: 3, name: "MacBook", price: 12999, category: "电脑", stock: 3 },
  { id: 4, name: "AirPods", price: 1299, category: "配件", stock: 50 },
  { id: 5, name: "键盘", price: 299, category: "配件", stock: 100 },
  { id: 6, name: "显示器", price: 2499, category: "电脑", stock: 8 },
  { id: 7, name: "鼠标", price: 99, category: "配件", stock: 200 },
];

// TS 链式调用
const result = products
  .filter(p => p.price > 100)
  .sort((a, b) => b.price - a.price)
  .slice(0, 5)
  .map(p => `${p.name}: ¥${p.price}`);
// → ["MacBook: ¥12999", "iPhone: ¥6999", "华为Mate: ¥5999", "显示器: ¥2499", "AirPods: ¥1299"]
```

### ☕ Java 完整代码

```java
// 文件名: Product.java
public class Product {
    private int id;
    private String name;
    private double price;
    private String category;
    private int stock;

    public Product(int id, String name, double price, String category, int stock) {
        this.id = id;
        this.name = name;
        this.price = price;
        this.category = category;
        this.stock = stock;
    }

    // getter
    public int getId() { return id; }
    public String getName() { return name; }
    public double getPrice() { return price; }
    public String getCategory() { return category; }
    public int getStock() { return stock; }
}
```

```java
// 文件名: StreamDemo.java
import java.util.Arrays;
import java.util.Comparator;
import java.util.List;
import java.util.stream.Collectors;

public class StreamDemo {
    public static void main(String[] args) {
        // 创建商品列表
        List<Product> products = Arrays.asList(
            new Product(1, "iPhone", 6999, "手机", 10),
            new Product(2, "华为Mate", 5999, "手机", 5),
            new Product(3, "MacBook", 12999, "电脑", 3),
            new Product(4, "AirPods", 1299, "配件", 50),
            new Product(5, "键盘", 299, "配件", 100),
            new Product(6, "显示器", 2499, "电脑", 8),
            new Product(7, "鼠标", 99, "配件", 200)
        );

        // ⭐ Stream API 链式调用
        List<String> result = products.stream()
            .filter(p -> p.getPrice() > 100)            // 筛选价格>100
            .sorted(Comparator.comparingDouble(Product::getPrice).reversed())  // 按价格降序
            .limit(5)                                    // 取前5个
            .map(p -> p.getName() + ": ¥" + p.getPrice()) // 映射成字符串
            .collect(Collectors.toList());               // 收尾成List

        // 打印结果
        result.forEach(System.out::println);
    }
}
```

```

iPhone: ¥6999.0
华为Mate: ¥5999.0
显示器: ¥2499.0
AirPods: ¥1299.0
```

## ❓ 练习1的疑问点 & 难点

### 🔴🔴 难点1（最核心）：`.sorted()` 需要传 `Comparator`

```java
// ❌ Java里不能直接 .sort() 无参排序对象列表！
// 因为Java不知道按什么规则排序对象（按价格？按名字？）

// ✅ 必须告诉它：按价格排序
.sorted(Comparator.comparingDouble(Product::getPrice))

// ✅ 降序：加 .reversed()
.sorted(Comparator.comparingDouble(Product::getPrice).reversed())
```

> 🧩 **前端类比**：
> ```typescript
> // TS/JS 你写 .sort((a, b) => b.price - a.price)
> // Java 你必须提供一个"比较器"（Comparator）
> ```

| 前端 `array.sort()` | Java `stream.sorted()` |
|---|---|
| `arr.sort()` — 字符串按字母排序 | `.sorted()` — 仅对 `String`/`Integer` 等有天然排序的类可用 |
| `arr.sort((a,b) => a.price - b.price)` | `.sorted(Comparator.comparingDouble(Product::getPrice))` |
| `arr.sort((a,b) => b.price - a.price)` | `.sorted(Comparator.comparingDouble(Product::getPrice).reversed())` |

### 🔴 难点2：`.limit(5)` 代替了 JS 的 `.slice(0, 5)`

```java
// JS: .slice(0, 5)    → 取索引0到5的元素
// Java: .limit(5)     → 取前5个元素
```

Java Stream 没有 `.slice()`，它的等价物：
- `.limit(n)` — 取前 n 个
- `.skip(n)` — 跳过前 n 个
- `.skip(2).limit(3)` — 从第3个开始取3个（≈ JS 的 `slice(2, 5)`）

### 🔴 难点3：结尾必须加 `.collect()` 收尾

```java
// ❌ 只写这些不会执行任何操作！
products.stream()
    .filter(p -> p.getPrice() > 100)
    .sorted(...);

// ✅ 必须加一个"终端操作"（terminal operation）来触发执行
List<String> result = products.stream()
    .filter(p -> p.getPrice() > 100)
    .sorted(...)
    .map(...)
    .collect(Collectors.toList());  // ← 终端操作，触发执行！
```

> 🧩 **前端类比**：Stream 是**惰性求值**的。就像 RxJS 的 Observable——你不 `.subscribe()`，管道里的操作都不会执行。`.collect()` ≈ `.subscribe()`。

### 🟡 避坑：`.forEach(System.out::println)` 是什么鬼？

```java
result.forEach(System.out::println);
```

| 写法 | 等价于 | 说明 |
|------|--------|------|
| `System.out::println` | `item -> System.out.println(item)` | 方法引用 |
| `Product::getName` | `p -> p.getName()` | 方法引用 |

> 🧩 **前端类比**：`::` ≈ 前端的箭头函数简写。`User::getName` ≈ `(u) => u.getName()`。**写熟了你会觉得比箭头函数还简洁！**

# 📝 练习2：用 `groupingBy` 实现按分类统计

> **难度：⭐⭐⭐ | 核心概念：`Collectors.groupingBy` ≈ lodash 的 `groupBy`**

## 🧩 前端类比

```typescript
// 前端 lodash 写法
import { groupBy } from 'lodash';

const grouped = groupBy(products, 'category');
// → {
//   "手机": [iPhone, 华为Mate],
//   "电脑": [MacBook, 显示器],
//   "配件": [AirPods, 键盘, 鼠标]
// }
```

## ☕ Java 完整代码

```java
// 文件名: GroupingByDemo.java
import java.util.*;
import java.util.stream.Collectors;

public class GroupingByDemo {
    public static void main(String[] args) {
        List<Product> products = Arrays.asList(
            new Product(1, "iPhone", 6999, "手机", 10),
            new Product(2, "华为Mate", 5999, "手机", 5),
            new Product(3, "MacBook", 12999, "电脑", 3),
            new Product(4, "AirPods", 1299, "配件", 50),
            new Product(5, "键盘", 299, "配件", 100),
            new Product(6, "显示器", 2499, "电脑", 8),
            new Product(7, "鼠标", 99, "配件", 200)
        );

        // ⭐ 1. 按分类分组 —— 最基础的用法
        Map<String, List<Product>> byCategory = products.stream()
            .collect(Collectors.groupingBy(Product::getCategory));

        System.out.println("=== 按分类分组 ===");
        byCategory.forEach((category, productList) -> {
            System.out.println(category + ": " + 
                productList.stream().map(Product::getName).collect(Collectors.joining(", ")));
        });

        // ⭐ 2. 按分类统计数量 —— 更实用！
        Map<String, Long> countByCategory = products.stream()
            .collect(Collectors.groupingBy(
                Product::getCategory,      // 按什么分组
                Collectors.counting()      // 每个组统计数量
            ));

        System.out.println("\n=== 各分类商品数量 ===");
        countByCategory.forEach((category, count) -> 
            System.out.println(category + ": " + count + " 个")
        );

        // ⭐ 3. 按分类统计价格总和 —— 超实用！
        Map<String, Double> totalPriceByCategory = products.stream()
            .collect(Collectors.groupingBy(
                Product::getCategory,
                Collectors.summingDouble(Product::getPrice)  // 求和
            ));

        System.out.println("\n=== 各分类商品总价 ===");
        totalPriceByCategory.forEach((category, total) -> 
            System.out.println(category + ": ¥" + total)
        );
    }
}
```

### ✅ 预期输出

```
=== 按分类分组 ===
电脑: MacBook, 显示器
手机: iPhone, 华为Mate
配件: AirPods, 键盘, 鼠标

=== 各分类商品数量 ===
电脑: 2 个
手机: 2 个
配件: 3 个

=== 各分类商品总价 ===
电脑: ¥15498.0
手机: ¥12998.0
配件: ¥1697.0
```

## ❓ 练习2的疑问点 & 难点

### 🔴 难点1：`Collectors.groupingBy` 的两种形态

```java
// 形态1：只分组，不统计
Map<String, List<Product>> map = products.stream()
    .collect(Collectors.groupingBy(Product::getCategory));

// 形态2：分组 + 下游统计
Map<String, Long> map = products.stream()
```

>

> 第二个参数 = 对每个分组做什么（你说的"分完以后算什么"）

### 🟡 避坑：`Map` 的遍历方式

```java
// Java 遍历 Map 的两种方式

// 方式1：forEach（最常用）
map.forEach((key, value) -> {
    System.out.println(key + ": " + value);
});

// 方式2：entrySet 遍历（和JS的 Object.entries() 很像）
for (Map.Entry<String, List<Product>> entry : map.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}
```

| JS | Java |
|---|---|
| `Object.entries(obj)` | `map.entrySet()` |
| `obj[key]` | `map.get(key)` |
| `Object.keys(obj)` | `map.keySet()` |

### 🟡 避坑：`Map` 打印出的顺序可能是乱的

```java
// HashMap 不保证顺序！
```

# 📝 练习3：写一个泛型方法 `<T> List<T> filterList(List<T> list, Predicate<T> p)`

> **难度：⭐⭐⭐⭐ | 核心概念：泛型方法 + 函数式接口 `Predicate`**

## 🧩 前端类比

```typescript
// 前端泛型 —— 你熟悉的
function filterList<T>(list: T[], predicate: (item: T) => boolean): T[] {
  return list.filter(predicate);
}

// 使用
const nums = [1, 2, 3, 4, 5];
const even = filterList(nums, n => n % 2 === 0);  // [2, 4]

const names = ["小明", "小红", "小刚"];
const withXiao = filterList(names, n => n.startsWith("小"));  // ["小明", "小红", "小刚"]
```

## ☕ Java 完整代码

```java
// 文件名: GenericMethodDemo.java
import java.util.*;
import java.util.function.Predicate;
import java.util.stream.Collectors;

public class GenericMethodDemo {

    // ⭐ 泛型方法定义
    // <T> 表示这是一个泛型方法，T 是类型参数
    // List<T> 返回值：返回一个 List，元素类型是 T
    // Predicate<T> p：一个"判断条件"，T → boolean 的函数
    public static <T> List<T> filterList(List<T> list, Predicate<T> p) {
        List<T> result = new ArrayList<>();
        for (T item : list) {
            if (p.test(item)) {    // p.test(item) ≈ JS 的 p(item)
                result.add(item);
            }
        }
        return result;
    }

    // ⭐ 进阶：用 Stream API 实现（一行搞定）
    public static <T> List<T> filterListStream(List<T> list, Predicate<T> p) {
        return list.stream()
            .filter(p)              // Predicate 可以直接传给 .filter()
            .collect(Collectors.toList());
    }

    public static void main(String[] args) {
        // ---------- 测试1：过滤整数 ----------
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5, 6);
        
        // 匿名内部类写法（像 JS 回调，但啰嗦）
        List<Integer> even = filterList(numbers, new Predicate<Integer>() {
            @Override
            public boolean test(Integer n) {
                return n % 2 == 0;
            }
        });
        System.out.println("偶数: " + even);  // [2, 4, 6]

        // Lambda 写法（推荐！像箭头函数）
        List<Integer> odd = filterList(numbers, n -> n % 2 != 0);
        System.out.println("奇数: " + odd);   // [1, 3, 5]

        // ---------- 测试2：过滤字符串 ----------
        List<String> names = Arrays.asList("小明", "小红", "小刚", "大明", "大壮");
        
        List<String> withXiao = filterList(names, name -> name.startsWith("小"));
        System.out.println("姓小的: " + withXiao);  // [小明, 小红, 小刚]

        // ---------- 测试3：过滤商品 ----------
        List<Product> products = Arrays.asList(
            new Product(1, "iPhone", 6999, "手机", 10),
            new Product(2, "键盘", 299, "配件", 100),
            new Product(3, "鼠标", 99, "配件", 200)
        );

        List<Product> cheap = filterList(products, p -> p.getPrice() < 500);
        cheap.forEach(p -> System.out.println("便宜商品: " + p.getName()));
        // 便宜商品: 键盘
        // 便宜商品: 鼠标
    }
}
```

## ❓ 练习3的疑问点 & 难点

### 🔴🔴 难点1（最核心）：`Predicate<T>` 是什么？

```java
// Predicate<T> ≈ (item: T) => boolean
// 它是一个"函数式接口"——只有一个抽象方法的接口

@FunctionalInterface
public interface Predicate<T> {
    boolean test(T t);   // 接收一个 T，返回 boolean
}
```

> 🧩 **前端类比**：
> 
> | 前端 | Java |
> |------|------|
> | `(item: T) => boolean` | `Predicate<T>` |
> | `(a: T, b: T) => number`（排序用） | `Comparator<T>` |
> | `(item: T) => void`（遍历用） | `Consumer<T>` |
> | `() => T`（提供值） | `Supplier<T>` |
> | `(T) => R`（转换用） | `Function<T, R>` |

**简单记忆法**：`Predicate` = 一个问问题的函数，返回 `true` 或 `false`。

### 🔴 难点2：泛型方法的语法 `<T>` 写在哪？

```java
// ❌ 错误写法
public List<T> filterList(List<T> list, Predicate<T> p) { }  // 编译报错：找不到 T

// ✅ 正确写法
public <T> List<T> filterList(List<T> list, Predicate<T> p) { }  // <T> 写在返回值前面
```

> 🧩 **前端类比**：
> ```typescript
> // TS 泛型函数
> function filterList<T>(list: T[], p: (item: T) => boolean): T[] { }
> ```
> 
> TS 的 `<T>` 写在函数名后面，**Java 的 `<T>` 写在返回值类型前面**！位置不同但意思一样。

### 🔴 难点3：匿名内部类 vs Lambda

```java
// 匿名内部类写法（Java 8 之前）
List<Integer> even = filterList(numbers, new Predicate<Integer>() {
    @Override
    public boolean test(Integer n) {
        return n % 2 == 0;
    }
});

// Lambda 写法（Java 8 之后，推荐！）
List<Integer> even = filterList(numbers, n -> n % 2 == 0);
```

> 🧩 **前端类比**：
>```typescript
>// 匿名内部类 ≈ 定义一个完整的类，只为了用一次
>// Lambda ≈ 箭头函数，简洁明了
>```
>
> **记住**：在 Java 里遇到任何"需要传一个函数"的地方，你都有两个选择：
> 1. 笨方式：`new SomeInterface() { @Override public ... }` 
> 2. 聪明方式：Lambda `参数 -> 表达式`

### 🟡 避坑：泛型方法 VS 泛型类

```java
// 方式A：泛型方法（推荐 - 你需要的）
public class GenericMethodDemo {
    public static <T> List<T> filterList(List<T> list, Predicate<T> p) { ... }
}

// 方式B：泛型类（也可以）
public class Filter<T> {
    public List<T> filterList(List<T> list, Predicate<T> p) { ... }
}
// 使用：new Filter<Integer>().filterList(...)
```

> **建议**：对于“某个方法需要泛型”的场景，用泛型方法（方式A）就够了，不需要把整个类搞成泛型类。

# 📝 练习4：体会类型擦除——打印 `List<String>` 和 `List<Integer>` 的 `getClass()`

> **难度：⭐ | 核心概念：Java泛型的"假"本质**

## 🧩 前端类比

```typescript
// TS 泛型——类型信息在运行时存在
function getType<T>(arr: T[]): string {
    return typeof arr[0];  // 可以获取元素类型
}

const strs: string[] = ["a", "b"];
const nums: number[] = [1, 2];

console.log(getType(strs));  // "string"
console.log(getType(nums));  // "number"
// → 类型信息在运行时保留！
```

## ☕ Java 完整代码

```java
// 文件名: TypeErasureDemo.java
import java.util.ArrayList;
import java.util.List;

public class TypeErasureDemo {

        // 创建两个"不同类型"的 ArrayList
        List<String> strings = new ArrayList<>();
        List<Integer> integers = new ArrayList<>();

        // 加入一些元素（证明它们能正常工作）
        strings.add("Hello");
        strings.add("World");

        // ⭐ 打印它们的 getClass() —— 看看"类型"是否相同！

    }
}
```

## ❓ 练习4的疑问点 & 难点

### 🔴🔴 难点1（最核心）：类型擦除 —— Java 泛型是"假的"

```java
List<String> strings = new ArrayList<>();
List<Integer> integers = new ArrayList<>();

// 运行时，strings 和 integers 都是 "class java.util.ArrayList"
// 泛型的 <String> 和 <Integer> 在编译后被删掉了！
```

> 🧩 **前端类比**：
> 
> **TS 泛型**：就像你在代码里写了完整的类型标注，`tsc` 编译后虽然也有类型擦除，但 TS 的类型信息**在编译时**被充分用来做类型检查，且有 `emitDecoratorMetadata` 等机制保留元数据。
>
> **Java 泛型**：更像是**编译时帮你检查类型正确性，然后把 `<T>` 的信息全删了**，运行时只知道它是 `ArrayList`，不知道它是 `ArrayList<String>` 还是 `ArrayList<Integer>`。
>
> **极度简化版**：
> - TS：`Array<string>` → 编译时检查 → 运行时保留部分信息
> - Java：`List<String>` → 编译时检查 → 运行时变成 `List`（删掉 `<String>`）

### 🔴 这个差异会带来什么后果？

```java
// ❌ 不能 new T()
public <T> T createInstance() {
    return new T();  // 编译报错！运行时不知道 T 是什么，无法创建
}

// ❌ 不能 instanceof T
public <T> void checkType(Object obj) {
    if (obj instanceof T) { }  // 编译报错！运行时 T 已被擦除
}

// ✅ 想实现上面的功能，需要传 Class<T> 参数
public <T> T createInstance(Class<T> clazz) throws Exception {
    return clazz.getDeclaredConstructor().newInstance();  // 反射
}
```

### 🟡 避坑：数组保留类型信息，集合不保留

```java
// 数组 —— 保留类型信息
String[] arr1 = new String[5];
Integer[] arr2 = new Integer[5];
arr1.getClass() != arr2.getClass()  // ✅ true（不同类型）

// 集合 —— 不保留类型信息
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
list1.getClass() == list2.getClass()  // ✅ true（相同类型！）
```

> **这就是为什么 Java 说"泛型数组是不安全的"**——数组和泛型的底层类型机制不兼容，这就是为什么你不能写 `new List<String>[10]`。

# 📝 练习5：用 `reduce` 实现一个自定义聚合操作

> **难度：⭐⭐⭐ | 核心概念：`reduce` 归约操作**

## 🧩 前端类比

```typescript
// JS reduce —— 你肯定写过
const numbers = [1, 2, 3, 4, 5];

// 求和
const sum = numbers.reduce((acc, cur) => acc + cur, 0);  // 15

// 求最大值
const max = numbers.reduce((acc, cur) => Math.max(acc, cur), 0);  // 5

// 拼接字符串
const joined = numbers.reduce((acc, cur) => acc + "-" + cur, "");  // "-1-2-3-4-5"
```

## ☕ Java 完整代码

```java
// 文件名: ReduceDemo.java
import java.util.Arrays;
import java.util.List;
import java.util.Optional;

public class ReduceDemo {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

        // ⭐ 1. 求和 —— 最基础的 reduce
        // reduce(初始值, (累加器, 当前元素) -> 新累加值)
        int sum = numbers.stream()
            .reduce(0, (acc, cur) -> acc + cur);
        System.out.println("sum = " + sum);  // 15

        // 简写版：Integer::sum 是内置的求和函数
        int sum2 = numbers.stream()
            .reduce(0, Integer::sum);
        System.out.println("sum2 = " + sum2); // 15

        // ⭐ 2. 求最大值
        // Optional 是因为如果列表为空，reduce 没有初始值时返回空
        Optional<Integer> max = numbers.stream()
            .reduce((acc, cur) -> acc > cur ? acc : cur);
        System.out.println("max = " + max.orElse(-1));  // 5

        // 简写版
        Optional<Integer> max2 = numbers.stream()
            .reduce(Integer::max);
        System.out.println("max2 = " + max2.get());     // 5

        // ⭐ 3. 求乘积
        int product = numbers.stream()
            .reduce(1, (acc, cur) -> acc * cur);
        System.out.println("product = " + product);     // 120 (1*2*3*4*5)

        // ⭐ 4. 拼接字符串
        String joined = numbers.stream()
            .map(String::valueOf)       // int → String
            .reduce("", (acc, cur) -> acc + "-" + cur);
        System.out.println("joined = '" + joined + "'"); // '-1-2-3-4-5'

        // ⭐ 5. 复杂案例：找出订单中总金额最高的订单
        List<Product> products = Arrays.asList(
            new Product(1, "iPhone", 6999, "手机", 10),
            new Product(2, "MacBook", 12999, "电脑", 3),
            new Product(3, "键盘", 299, "配件", 100)
        );

        // 找出价格最高的商品
        Optional<Product> mostExpensive = products.stream()
            .reduce((p1, p2) -> p1.getPrice() > p2.getPrice() ? p1 : p2);
        
        mostExpensive.ifPresent(p -> 
            System.out.println("最贵的商品: " + p.getName() + " ¥" + p.getPrice())
        );  // 最贵的商品: MacBook ¥12999.0
    }
}
```

## ❓ 练习5的疑问点 & 难点

### 🔴 难点1：`reduce` 的两种形态

```java
// 形态1：有初始值 —— 返回确定类型（不会为空）
T reduce(T identity, BinaryOperator<T> accumulator);

// 形态2：无初始值 —— 返回 Optional（可能为空）
Optional<T> reduce(BinaryOperator<T> accumulator);
```

| JS | Java |
|---|---|
| `arr.reduce((acc, cur) => acc + cur, 0)` | `.reduce(0, (acc, cur) -> acc + cur)` |
| `arr.reduce((acc, cur) => Math.max(acc, cur))` | `.reduce((acc, cur) -> acc > cur ? acc : cur)` |

**区别**：Java 无初始值时返回 `Optional<T>`，需要处理"列表为空"的情况！

### 🔴 难点2：`BinaryOperator<T>` 是啥？

```java
// BinaryOperator<T> = (T, T) -> T
// 接收两个同类型的值，返回一个同类型的值

@FunctionalInterface
public interface BinaryOperator<T> extends BiFunction<T, T, T> {
    // 继承 BiFunction 的 apply 方法
    T apply(T t1, T t2);
}
```

> 🧩 **前端类比**：`BinaryOperator` ≈ `(a: T, b: T) => T`，就像 `Math.max`、`(a, b) => a + b` 这种"两个输入一个输出"的函数。

### 🟡 避坑：`Optional` 的 `orElse` 和 `get`

```java
// 如果有值，返回该值；否则返回默认值
max.orElse(-1)      // ✅ 安全的取值方式

// 如果有值，返回该值；否则抛异常
max.get()           // ❌ 如果为空会抛 NoSuchElementException
max.orElseThrow(...) // ✅ 比 get() 好，可以自定义异常
```

> **规则**：碰到 `Optional`，永远优先用 `orElse`/`orElseThrow`，而不是 `get()`。

# 💡 前端转Java的Day 3定心丸

```java
            new Product(4, "AirPods", 1299, "配件", 50),
            new Product(5, "键盘", 299, "配件", 100),
            new Product(6, "显示器", 2499, "电脑", 8),
            new Product(7, "鼠标", 99, "配件", 200)
        );

        Map<String, List<Product>> byCategory = products.stream()
            .collect(Collectors.groupingBy(Product::getCategory));

        byCategory.forEach((category, productList) -> {
            System.out.println(category + ": " + productList);
        });
    }
}
```

## 💡 为什么跟你前端直觉不一样？

```typescript
// 前端——你不需要构造器也能创建对象
const product = {
  id: 1,
  name: "iPhone",
  price: 6999,
  category: "手机",
  stock: 10
};  // ✅ 直接对象字面量就搞定了！
```

```java
// Java——必须通过构造器
Product p = new Product(1, "iPhone", 6999, "手机", 10);
//           ↑ 必须 Product 类里有这个构造器！
//           否则编译器不知道：5个参数分别赋值给哪些字段？
```

> **核心差异**：前端有"对象字面量"（`{}` 一把梭），Java 没有！Java 创建对象必须先有类 + 构造器，告诉编译器"参数1 → 字段A，参数2 → 字段B..."。
