# 🗓️ Week 1：Java 语言核心——"把 TypeScript 翻译成 Java 方言"

> **目标**：用7天完成Java语法的"概念映射"，能写出类型安全、结构清晰的Java代码
> **对标前端心智**：从"JS/TS自由风格"切换到"Java强约束风格"

## Day 1：类型系统 + 基础语法（4h）

| 前端概念 | Java对应 | 关键差异 |
|----------|----------|----------|
| `let/const` 动态类型 | `int/String/var` 静态强类型 | 🔴 Java是**刚性**类型，编译时确定 |
| TS `interface` | Java `interface` | 🟢 同名但Java的interface不含实现（default方法除外） |
| TS `type` 联合类型 | ❌ Java无直接对应 | 🟡 用sealed class + pattern matching替代 |
| `console.log()` | `System.out.println()` | — |

**练习**：用Java重写一个你之前写的TS工具函数库（deepClone → 用序列化思想理解、debounce → 无直接对应但理解范式差异）

## Day 2：类与对象——"React Class Component 的究极形态"（4h）

| 前端概念 | Java对应 |
|----------|----------|
| JS `class` | Java `class`（但强约束100倍） |
| `constructor()` | Java 构造函数 + 重载 |
| `this` 指向 | Java `this`（永远指向实例，无绑定问题） |
| `#privateField` | `private` 关键字 |
| `static` 方法 | `static` 方法（语义一致） |
| TS `readonly` | `final` 关键字 |

**练习**：创建一个完整的实体类体系 → 比如 `User → Order → OrderItem`，体会封装/继承/多态

## Day 3：泛型 + 集合框架（5h）⭐核心日

| 前端概念 | Java对应 |
|----------|----------|
| TS `Array<T>` | `List<T>` / `ArrayList<T>` |
| TS `Record<K,V>` / `Map` | `Map<K,V>` / `HashMap<K,V>` |
| TS `Set<T>` | `Set<T>` / `HashSet<T>` |
| TS 泛型 `<T extends ...>` | Java 泛型 `<T extends ...>`（语义几乎一致） |
| JS `[...arr].map().filter()` | Stream API `.map().filter().collect()` |

**练习**：用Stream API重写你用数组方法链处理数据的一个复杂场景

## Day 4：异常 + Optional + 空安全（3h）

| 前端概念 | Java对应 |
|----------|----------|
| `try/catch/finally` | 同名（但Java是**检查型异常**） |
| `throw new Error()` | `throw new Exception()` |
| `undefined` / `null` 判断 | `Optional<T>` |
| TS `?.` 可选链 | `Optional.map()` / `Optional.flatMap()` |

**练习**：体会Java的"强迫你处理异常"哲学 vs JS的"崩了就崩了"哲学

## Day 5：注解 + 反射（4h）

| 前端概念 | Java对应 |
|----------|----------|
| TS `@decorator` | Java `@Annotation`（更强大，编译期+运行期） |
| `Proxy` / `Reflect` | Java Reflection API |
| `Object.keys()` 遍历属性 | `Class.getDeclaredFields()` |

**练习**：写一个自定义注解，用反射读取——理解Spring Boot的"魔法"是如何实现的

## Day 6：Java 21 现代特性速览（4h）

- **Record** = 不可变DTO，对标TS的 `readonly` interface + 自动生成constructor/getter
- **Pattern Matching** = 对标TS的 discriminated union + switch
- **Virtual Threads** = 对标Node.js的异步模型（但更底层透明）
- **Text Blocks** = Java版的模板字符串 `"""..."""`

**练习**：用Record改写所有DTO，用Pattern Matching处理多态逻辑

## Day 7：综合实战 + 阶段性复盘（6h）

**大作业**：纯Java（不用任何框架）实现一个控制台版"简易REST API路由器"
- 体会没有Spring Boot时手写反射路由的痛
- 为Week 2的"Spring Boot真香"做心理铺垫

# 🗓️ Week 2：Spring Boot 核心——"Java 界的 Next.js"

> **目标**：掌握Spring Boot分层架构、REST API开发、数据库操作
> **对标前端心智**：Spring Boot = Next.js/Nuxt（全栈框架），但分层更严格

## Day 8：IOC/DI + Spring Boot启动流程（5h）⭐关键日

| 前端概念 | Java对应 |
|----------|----------|
| React `Context.Provider` | Spring IOC容器 |
| `useContext()` / 依赖注入 | `@Autowired` / 构造器注入 |
| Next.js 约定式路由 | Spring Boot 自动配置（Auto Configuration） |
| `npm run dev` 热更新 | Spring DevTools |
| `package.json` | `pom.xml` / `build.gradle` |

**练习**：手写一个迷你IOC容器（≈200行），彻底理解控制反转

## Day 9：Controller + REST API（4h）

| 前端概念 | Java对应 |
|----------|----------|
| Next.js API Route | `@RestController` + `@GetMapping` |
| API Route 的 `req.body` | `@RequestBody` DTO |
| API Route 的 `req.query` | `@RequestParam` |
| 动态路由 `[id]` | `@PathVariable` |
| Express 中间件 | Spring Interceptor / Filter |

**练习**：用Spring Boot重写你之前Node.js项目的一组CRUD接口

## Day 10：Service层 + 分层架构（4h）

| 前端概念 | Java对应 |
|----------|----------|
| `src/services/api.ts` | `@Service` 层 |
| DTO（前端定义的数据结构） | Java DTO/VO/Entity 三层分离 |
| `zod` 校验 | `@Valid` + `jakarta.validation` |
| 全局错误处理中间件 | `@RestControllerAdvice` |

**练习**：完整实现 Controller → Service → Repository 三层调用链

## Day 11：MyBatis-Plus 数据库操作（5h）

| 前端概念 | Java对应 |
|----------|----------|
| Prisma / TypeORM | MyBatis-Plus / Spring Data JPA |
| Prisma Schema | Entity类 + `@Table` 注解 |
| `prisma.user.findMany()` | `userMapper.selectList()` |
| Prisma Migration | Flyway / Liquibase |
| SQL直接写 | MyBatis XML / 注解SQL |

**推荐选型**：MyBatis-Plus（对前端更友好，SQL可控，接近你写Prisma的感觉）

## Day 12：Spring Data JPA 对比 + 选型（3h）

快速对比 JPA 和 MyBatis，理解两者的设计哲学差异（类比 Prisma vs Knex.js），最终选择一个深入。

**练习**：分别用JPA和MyBatis实现同一张表的CRUD

## Day 13：事务管理 + 文件上传 + 参数校验（4h）

| 前端概念 | Java对应 |
|----------|----------|
| 数据库事务（Prisma `$transaction`） | `@Transactional` |
| `multer` 文件上传 | `MultipartFile` |
| `zod` / `yup` 校验 | `@Valid` + Annotation校验 |

## Day 14：Week 2 综合实战（8h）🔥

**大作业**：实现一个完整的"后台管理系统后端"
- 用户管理（CRUD + 分页 + 条件查询）
- 文件上传（头像）
- 统一返回格式 `Result<T>`
- 全局异常处理
- 参数校验

# ⚠️ 前端转Java的7个关键心态转换

| # | 前端习惯 | Java现实 |
|---|----------|----------|
| 1 | `npm install` 秒装 | Maven下载可能很慢（提前配国内镜像！） |
| 2 | 改了代码立刻看到 | 编译+重启有延迟（DevTools缓解但不能消除） |
| 3 | JSON一把梭 | 必须定义Entity/DTO/VO/BO多层对象 |
| 4 | 单线程事件循环 | 多线程模型，要关注并发安全 |
| 5 | `any` 随意逃生 | 没有`any`，泛型必须写清楚 |
| 6 | 启动快（秒级） | Spring Boot启动可能30s+（习惯就好） |
| 7 | 函数式灵活 | OOP强约束，但设计模式是双刃剑 |
